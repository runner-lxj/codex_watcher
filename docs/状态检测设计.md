# Codex Watcher 状态检测设计文档

## 1. 状态定义

| 状态 | LED 行为 | 含义 |
|------|----------|------|
| RUNNING | 绿灯常亮 | 用户发送命令后，Codex 正在调用模型/执行工具 |
| WAITING | 黄灯闪烁(200ms) | 需要用户手动授权，持续闪烁直到用户同意或拒绝 |
| DONE | 红灯常亮 | 本轮交互结果已展示给用户，等待用户发起新命令 |
| ERROR | 红黄绿三灯闪烁(200ms) | 超时/网络错误/API错误/进程崩溃 |

## 3. 检测逻辑

### 3.1 RUNNING（绿灯常亮）

**触发条件（满足任一即判定）:**
1. esponse.created > response.completed → 模型调用进行中
2. 用户在10秒内发送了命令 + 进程在 → 等待模型响应
3. 	okens_used 在两次查询间有增长 → 正在执行工具

**退出条件:** 模型调用完成 + token 不再增长 + 用户命令超过10秒 → DONE

### 状态规则

- DONE 之后不再触发 WAITING：红灯亮起后，黄灯不会闪烁
- WAITING 只在 RUNNING 期间触发：模型调用中检测到权限请求才会亮黄灯

### 3.2 WAITING（黄灯闪烁，粘性状态）

**进入条件（全部满足）:**
1. esponse.output_item.done 事件中 	ype:function_call 包含 sandbox_permissions
2. pproval_mode == "on-request"
3. 响应已完成（created <= completed）
4. 响应完成至少5秒（排除正常工具执行，正常1-2秒）
5. 权限事件在响应完成之前或同时（perm <= completed）
6. 最近有用户输入（
ow - inp <= IDLE_TIMEOUT）
7. 启动后的新事件（completed >= started_at）
8. 非冷却期（退出 WAITING 后30秒内不重复触发）\n9. 当前状态不是 DONE（DONE 之后不再触发 WAITING）

**保持条件（一旦进入）:**
- 持续闪烁，直到：
  - 新响应开始（用户点了 Yes）→ 切 RUNNING
  - 新响应完成（用户点了 No）→ 切 DONE
  - 15秒超时 → 切 DONE

**退出后:** 设置30秒冷却期，期间不再触发 WAITING

### 3.3 DONE（红灯常亮）

**触发条件:**
1. 进程不存在 → DONE
2. 进程存在但不满足 RUNNING / WAITING / ERROR 条件 → DONE

### 3.4 ERROR（红黄绿三灯闪烁）

**触发条件（纯 watchdog）:**
- 进程存在但日志超过 WATCHDOG_TIMEOUT (15秒) 无任何更新

## 6. WAITING 检测的已知问题

### 误判原因

sandbox_permissions 是 shell_command 工具的**参数定义**，不是工具调用的参数值。模型生成工具调用时，参数中只包含 command，不包含 sandbox_permissions。但 sandbox_permissions 关键词出现在工具定义文本中，每次响应都会匹配。

Codex 的授权决策（弹不弹窗）在收到工具调用后、执行前做出，**不写入日志**。从日志无法区分"需要授权的工具调用"和"不需要授权的工具调用"。

### 已尝试的检测方案

| 方案 | 失败原因 |
|------|----------|
| 匹配 sandbox_permissions 关键词 | 每次响应都匹配（工具定义文本） |
| 匹配 esponse.output_item.done + sandbox_permissions | 同上 |
| 匹配 	ype:function_call + sandbox_permissions | 同上 |
| 超时检测（响应完成后暂停 N 秒） | 无法区分"等待授权"和"正常工具执行间隔" |
| watchdog（日志静默超时） | websocket 重试消息持续更新 last_activity |

### 缓解措施

- **5秒暂停阈值**：正常工具执行1-2秒，只有真正等授权才会暂停5秒以上
- **30秒冷却期**：WAITING 退出后30秒内不重复触发
- **15秒超时**：WAITING 最多持续15秒，避免一直闪

## 7. 实现要点

- 数据库使用 mode=ro（不能用 immutable=1，会导致 WAL 数据延迟约20秒）
- 错误检测使用纯 watchdog，不匹配文本关键词（模型推理内容会误匹配）
- 权限检测匹配 esponse.output_item.done + 	ype:function_call + sandbox_permissions 三重条件
- WAITING 需要响应暂停至少5秒才触发
- WAITING 为粘性状态，进入后持续到用户响应或15秒超时
- WAITING 退出后30秒冷却期
- RUNNING 在用户发命令后10秒内触发
- 多会话取最近活跃的会话
- LED 引脚: 绿=GPIO13, 黄=GPIO12, 红=GPIO14
- 无蜂鸣器和电位器
- 硬件端固件无需改动
