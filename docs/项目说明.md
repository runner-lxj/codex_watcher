# Codex Watcher 项目文档

## 1. 项目概述

Codex Watcher 是一个基于 ESP32 的硬件监控设备，用于实时指示 Codex 的工作状态。通过 LED 灯和蜂鸣器的组合，用户无需查看屏幕即可掌握 Codex 的运行情况。

### 目标用户

使用 Codex（OpenAI 编程助手）进行长时间开发任务的工程师，需要在 Codex 运行期间做其他事情，同时关注其状态。

---

## 2. 系统架构

```
┌─────────────┐    USB/串口     ┌─────────────┐
│  主机 (PC)   │  ──────────>  │  ESP32 开发板 │
│             │   115200 baud  │             │
│ codex_monitor│                │   ┌───────┐ │
│    .py      │   串口指令      │   │ main  │ │
│             │                │   │ .cpp  │ │
│ 状态检测     │                │   └───┬───┘ │
│ (进程/日志)  │                │       │     │
└─────────────┘                └───────┼─────┘
                                       │
                              ┌────────┼────────┐
                              │        │        │
                           LED×3    蜂鸣器    电位器
                          红黄绿    有源      控制音量
```

### 工作流程

1. **主机端** `codex_monitor.py` 持续检测 Codex 的运行状态
2. 检测到状态变化时，通过串口发送对应指令到 ESP32
3. **ESP32 固件**接收指令，控制 LED 和蜂鸣器执行相应的灯效

---

## 3. 状态灯效定义

| 状态 | 串口指令 | 绿灯 | 黄灯 | 红灯 | 蜂鸣器 | 触发场景 |
|------|----------|------|------|------|--------|----------|
| 空闲 | `IDLE` | OFF | OFF | OFF | OFF | Codex 未启动或已关闭 |
| 执行中 | `RUNNING` | 常亮 | OFF | OFF | OFF | Codex 正在执行代码/思考 |
| 等待权限 | `WAITING` | OFF | 闪烁(200ms) | OFF | 间歇响 | Codex 需要用户确认权限 |
| 工作结束 | `DONE` | OFF | OFF | 常亮 | 长响 | Codex 任务完成 |
| 连接异常 | `ERROR` | 闪烁(200ms) | 闪烁(200ms) | 闪烁(200ms) | 快响 | 模型连接失败/响应超时 |

## 4. 硬件需求

### 元器件清单

| 元器件 | 数量 | 说明 |
|--------|------|------|
| ESP32 开发板 | 1 | ESP32-WROOM-32 或类似型号 |
| LED（绿/黄/红） | 各1 | 5mm 直插或贴片均可 |
| 220Ω 电阻 | 3 | LED 限流电阻 |
| 有源蜂鸣器 | 1 | 通电即响，非无源 |
| 电位器 | 1 | 10KΩ，用于调节蜂鸣器音量 |
| 面包板/杜邦线 | 若干 | 接线用 |

### 接线图

```
ESP32 引脚          元器件
─────────────────────────────
GPIO 2   ───[220Ω]─── 绿灯 LED(+) ──── GND
GPIO 4   ───[220Ω]─── 黄灯 LED(+) ──── GND
GPIO 16  ───[220Ω]─── 红灯 LED(+) ──── GND
GPIO 17  ──────────── 蜂鸣器(+) ──────── GND
GPIO 34  ──────────── 电位器(中间脚)
3.3V     ──────────── 电位器(一端)
GND      ──────────── 电位器(另一端)
```

**注意事项：**
- GPIO 34 是只读 ADC1 引脚，只能用于输入（读取电位器值），不能驱动 LED
- ESP32 GPIO 编号直接印在芯片上，不要和 Arduino 的 D0/D1 标记混淆
- LED 必须串联限流电阻，否则可能烧毁

## 5. 需要修改的配置参数

### 5.1 引脚配置（必须修改）

打开 `firmware/src/main.cpp`，修改顶部的 `#define`：

```cpp
#define PIN_GREEN_LED     2      // 改成你绿灯实际接的 GPIO
#define PIN_YELLOW_LED    4      // 改成你黄灯实际接的 GPIO
#define PIN_RED_LED       16     // 改成你红灯实际接的 GPIO
#define PIN_BUZZER        17     // 改成你蜂鸣器实际接的 GPIO
#define PIN_POTENTIOMETER 34     // 改成你电位器实际接的 GPIO
```

### 5.2 蜂鸣器参数（按需修改）

```cpp
#define BUZZER_FREQ  2000  // 蜂鸣器频率(Hz)
                           // 有源蜂鸣器：此值影响不大，保持默认
                           // 无源蜂鸣器：改成蜂鸣器标称频率
```

### 5.3 闪烁速度（可选修改）

```cpp
#define BLINK_FAST_MS  200  // 快闪间隔(ms)，WAITING/ERROR 状态使用
#define BLINK_SLOW_MS  500  // 慢闪间隔(ms)，当前未使用，预留
```

### 5.4 主机端检测配置（可选修改）

`host/codex_monitor.py` 中的关键变量：

```python
CODEX_LOG_DIR = "E:\\workspace\\codex_logs"  # Codex 日志目录，改成你的实际路径
CODEX_ERROR_KEYWORDS = ["timeout", "error", "connection failed", "rate limit"]
CODEX_PERMISSION_KEYWORDS = ["permission", "approve", "confirm", "allow"]
```

## 6. 环境搭建与部署

### 6.1 固件编译与烧录

**前提：** 安装 [PlatformIO](https://platformio.org/install)（推荐通过 VS Code 插件）

```powershell
cd firmware

# 编译
pio run

# 编译并烧录到 ESP32
pio run -t upload

# 串口监控（查看 ESP32 输出）
pio device monitor
```

**手动安装 PlatformIO CLI：**
```powershell
pip install platformio
```

### 6.2 主机端脚本

**前提：** 安装 Python 3.8+ 和 pyserial

```powershell
pip install pyserial

# 查看可用串口
python -m serial.tools.list_ports

# 手动测试模式（推荐先用这个验证接线）
python host/codex_monitor.py --port COM3 --mode manual

# 自动监控模式
python host/codex_monitor.py --port COM3

# 自动检测串口（不指定端口）
python host/codex_monitor.py
```

## 7. 串口通信协议

### 基本信息

- 波特率：115200
- 数据格式：8N1（8 数据位，无校验，1 停止位）
- 行尾：`\n`

### 指令列表（主机 -> ESP32）

| 指令 | 说明 | 响应 |
|------|------|------|
| `RUNNING` | 设置为执行中状态 | `OK:RUNNING` |
| `WAITING` | 设置为等待权限状态 | `OK:WAITING` |
| `DONE` | 设置为工作结束状态 | `OK:DONE` |
| `ERROR` | 设置为连接异常状态 | `OK:ERROR` |
| `IDLE` | 设置为空闲状态 | `OK:IDLE` |
| `STATUS` | 查询当前状态和音量 | `STATE:RUNNING,VOL:75` |
| `PING` | 心跳检测 | `PONG` |

### 响应格式

- 成功响应：`OK:状态名`
- 错误响应：`ERR:UNKNOWN_CMD:指令内容`
- 状态查询：`STATE:状态名,VOL:音量百分比`

### 调试示例

使用串口调试工具（如 PuTTY、串口助手）直接发送：

```
PING           -> PONG
STATUS         -> STATE:IDLE,VOL:50
RUNNING        -> OK:RUNNING
STATUS         -> STATE:RUNNING,VOL:50
```

## 8. 故障排查

### ESP32 不响应

1. 检查 USB 线是否支持数据传输（部分线只能充电）
2. 检查设备管理器中是否识别到 COM 端口
3. 按住 ESP32 的 BOOT 按钮再插 USB，尝试进入下载模式
4. 确认 PlatformIO 选择了正确的板型（ESP32 Dev Module）

### LED 不亮

1. 确认 GPIO 编号是否正确（代码中的值和实际接线一致）
2. 确认 LED 极性（长脚 +，短脚 -）
3. 确认电阻值（220Ω-1KΩ 均可）
4. 用万用表测量 GPIO 输出是否有高低电平变化

### 蜂鸣器不响

1. 确认是有源蜂鸣器（通电就响）还是无源蜂鸣器（需要 PWM）
2. 确认蜂鸣器正负极接反了吗
3. 尝试在手动模式下发送 `DONE` 指令测试

### 电位器读数不准

1. 确认接的是 GPIO 32-39（ADC1），GPIO 0-31 不支持模拟读取
2. 检查电位器两端是否分别接了 3.3V 和 GND
3. 发送 `STATUS` 指令查看当前音量值，旋转电位器观察变化

### 串口连接失败

1. 确认 `pip install pyserial` 已安装（注意不是 `serial`）
2. 确认端口号正确：`python -m serial.tools.list_ports`
3. 确认没有其他程序占用该串口

## 10. 后续扩展方向

- **WebSocket 模式**：ESP32 连接 WiFi，通过 WebSocket 接收状态，摆脱 USB 线
- **更多状态**：增加 Codex 的 token 消耗、任务进度等信息显示
- **OLED 显示屏**：在 ESP32 上接一个小屏幕，显示当前状态文字
- **多设备联动**：多台电脑的 Codex 共享同一个监控器
